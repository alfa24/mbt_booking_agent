# Архитектурные решения (ADR)

## Принципы ведения ADR

- Каждое значимое архитектурное решение фиксируется отдельным документом
- ADR имеет постоянный номер и не удаляется — только дополняется или заменяется новым
- Статусы: `Proposed` → `Accepted` / `Rejected` / `Superseded`
- Документ должен содержать контекст, рассмотренные альтернативы, решение и последствия

## Шаблон ADR

```markdown
# ADR-XXX: Название решения

## Статус
Proposed | Accepted | Rejected | Superseded by ADR-YYY

## Контекст
Описание проблемы и ограничений

## Рассмотренные варианты

### Вариант 1
**За:** ...
**Против:** ...

### Вариант 2
...

## Решение
Выбранный вариант с обоснованием

## Последствия
Что изменилось после принятия решения

## Ссылки
Связанные документы
```
