# Дорожная карта развития системы

## Организация работ

Этот документ определяет стратегические этапы развития системы бронирования загородного жилья. Каждый этап — это самостоятельный инкремент продукта, фиксируемый в отдельном `tasklist`. Принцип «один этап — один tasklist» обеспечивает чёткую трассировку от целей к результатам. Tasklist'ы располагаются в `docs/tasks/` и содержат детальное планирование итераций внутри этапа.

## Ключевые особенности плана

**Backend-first развитие**
Бизнес-логика концентрируется в ядре системы. Клиенты (Telegram-бот, веб-приложение) — только интерфейсы к единому API. Это позволяет добавлять новые каналы взаимодействия без дублирования логики.

**Инкрементальная эволюция**
Каждый этап поставляет работающий продукт с измеримой ценностью. Нет «больших взрывов» — только последовательное наращивание функциональности от MVP к полноценной платформе.

**Параллелизация по доменам**
После выделения backend API работа над клиентами (бот, веб) может вестись параллельно. Доменная модель (бронирования, тарифы, расходники) стабилизируется на ранних этапах.

## Детальное описание этапов

### Этап 0: MVP Telegram-бот

**Ценность:** Работающий бот, через который можно бронировать дома текстовыми сообщениями.

**Критерии завершения:**
- Пользователь может забронировать дом естественным языком
- Система понимает даты, количество гостей, выбор дома
- Арендодатель видит список бронирований
- Данные хранятся в памяти (in-memory)

---

### Этап 1: Backend API и база данных

**Ценность:** Надёжное хранение данных, возможность подключать новых клиентов к единому API.

**Критерии завершения:**
- Backend API с эндпоинтами для бронирований, домов, пользователей
- PostgreSQL для хранения данных
- Бот работает через API, in-memory хранение удалено
- Миграция существующих бронирований (если есть)

---

### Этап 2: Веб-приложение для арендаторов

**Ценность:** Полноценный личный кабинет с историей бронирований, календарём, уведомлениями.

**Критерии завершения:**
- Регистрация и вход через Telegram OAuth
- Просмотр доступных домов и календаря занятости
- Создание и управление бронированиями через UI
- История поездок и заметки о расходниках

---

### Этап 3: Панель управления арендодателя

**Ценность:** Полноценный инструмент управления бизнесом без привязки к Telegram.

**Критерии завершения:**
- Управление календарём доступности (закрытие/открытие дат)
- Управление тарифами для типов гостей
- Просмотр всех бронирований с фильтрами
- Учёт расходников в домах

---

### Этап 4: Интеграции и автоматизация

**Ценность:** Автономная работа системы без ручного контроля, монетизация.

**Критерии завершения:**
- Интеграция платёжной системы (предоплата бронирований)
- Уведомления о бронированиях
- Circuit breaker для LLM-провайдера
- Резервный LLM-провайдер на случай сбоев
