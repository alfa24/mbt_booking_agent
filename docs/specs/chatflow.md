# Chat Flow — Сценарии использования чата

## Обзор

Документ описывает сценарии взаимодействия пользователей с чат-ассистентом в системе бронирования загородного жилья. Чат доступен всем авторизованным пользователям и поддерживает текстовый и голосовой ввод/вывод.

---

## Общие возможности чата

### Интерфейс

- **Маршрут:** `/chat`
- **Доступ:** Все авторизованные пользователи (tenant, owner, both)
- **Тип:** Полноэкранный чат без списка диалогов (один чат на пользователя)

### Функциональность

| Функция | Описание |
|---------|----------|
| Текстовый ввод | Многострочное поле для сообщений |
| Голосовой ввод | Web Speech API для распознавания речи |
| Голосовой вывод | Text-to-Speech для сообщений ассистента |
| История | Постоянное хранение истории сообщений |
| Intent-обработка | Распознавание намерений и выполнение действий |

---

## Сценарии для арендатора (Tenant)

### 1. Бронирование через чат

**Описание:** Арендатор бронирует дом, отправляя естественноязыковое сообщение.

**Примеры запросов:**
- "Хочу забронировать Старый дом на выходные"
- "Забронируй Новый дом с 15 по 20 июня, будем 2 взрослых и ребёнок"
- "Есть ли свободные даты в Старом доме на следующую неделю?"

**Процесс:**
1. Пользователь отправляет сообщение с намерением забронировать
2. Ассистент распознаёт intent `create_booking`
3. Извлекаются параметры: дом, даты, состав группы
4. При недостающих данных — уточняющие вопросы
5. Подтверждение деталей бронирования
6. Выполнение бронирования через API
7. Ответ с подтверждением и деталями

**Результат:** Созданное бронирование + уведомление в чате

---

### 2. Просмотр своих бронирований

**Описание:** Арендатор запрашивает список своих бронирований.

**Примеры запросов:**
- "Покажи мои бронирования"
- "Какие у меня предстоящие поездки?"
- "Когда моё следующее бронирование?"

**Процесс:**
1. Пользователь отправляет запрос
2. Ассистент распознаёт intent `list_bookings`
3. Запрос к API: `GET /bookings?tenant_id={user_id}`
4. Форматирование ответа со списком

**Результат:** Список бронирований с датами, статусами и суммами

---

### 3. Отмена бронирования

**Описание:** Арендатор отменяет бронирование через чат.

**Примеры запросов:**
- "Отмени моё бронирование на 15 июня"
- "Хочу отменить поездку в Старый дом"
- "Отмени последнее бронирование"

**Процесс:**
1. Пользователь запрашивает отмену
2. Ассистент распознаёт intent `cancel_booking`
3. Поиск бронирования по контексту/датам
4. Подтверждение отмены
5. Выполнение: `PATCH /bookings/{id}` с `status: cancelled`
6. Уведомление об успешной отмене

**Результат:** Бронирование отменено, даты освобождены

---

### 4. Просмотр доступных домов

**Описание:** Арендатор узнаёт о доступных домах и их характеристиках.

**Примеры запросов:**
- "Какие дома доступны?"
- "Расскажи про Старый дом"
- "Какая вместимость у Нового дома?"

**Процесс:**
1. Запрос информации о домах
2. Ассистент распознаёт intent `list_houses` или `get_house_info`
3. Запрос к API: `GET /houses` или `GET /houses/{id}`
4. Форматирование ответа с описанием

**Результат:** Информация о домах, вместимости, описании

---

### 5. Проверка календаря занятости

**Описание:** Арендатор проверяет свободные даты для дома.

**Примеры запросов:**
- "Когда свободен Старый дом?"
- "Есть ли места на Новый год?"
- "Покажи календарь Нового дома на декабрь"

**Процесс:**
1. Запрос календаря
2. Ассистент распознаёт intent `check_calendar`
3. Запрос к API: `GET /houses/{id}/calendar`
4. Форматирование занятых/свободных дат

**Результат:** Визуализация или список свободных дат

---

### 6. Фиксация результатов поездки

**Описание:** После выезда арендатор фиксирует фактические данные.

**Примеры запросов:**
- "Зафиксируй результаты поездки"
- "Мы были 3 взрослых вместо 2"
- "Добавь заметку о проживании"

**Процесс:**
1. Запрос на фиксацию
2. Ассистент распознаёт intent `record_stay`
3. Запрос фактического состава группы
4. Обновление: `PATCH /bookings/{id}` с `guests_actual`
5. Пересчёт итоговой суммы

**Результат:** Данные зафиксированы, сумма пересчитана

---

## Сценарии для арендодателя (Owner)

### 7. Просмотр всех бронирований

**Описание:** Арендодатель запрашивает список всех бронирований.

**Примеры запросов:**
- "Покажи все бронирования"
- "Какие бронирования на этой неделе?"
- "Сколько подтверждённых бронирований в Старом доме?"

**Процесс:**
1. Запрос списка
2. Ассистент распознаёт intent `list_all_bookings`
3. Запрос к API: `GET /bookings` с фильтрами
4. Форматирование ответа

**Результат:** Список бронирований с фильтрами по дому/дате/статусу

---

### 8. Подтверждение бронирования

**Описание:** Арендодатель подтверждает ожидающее бронирование.

**Примеры запросов:**
- "Подтверди бронирование на 15 июня"
- "Подтверди бронь Иванова"
- "Подтверди последнее бронирование"

**Процесс:**
1. Запрос на подтверждение
2. Ассистент распознаёт intent `confirm_booking`
3. Поиск бронирования
4. Выполнение: `PATCH /bookings/{id}` с `status: confirmed`
5. Уведомление об успехе

**Результат:** Бронирование подтверждено

---

### 9. Управление календарём доступности

**Описание:** Арендодатель блокирует или разблокирует даты.

**Примеры запросов:**
- "Заблокируй Старый дом на 25-30 декабря"
- "Открой даты в Новом доме с 1 января"
- "Сделай недоступным Старый дом на выходные"

**Процесс:**
1. Запрос на изменение доступности
2. Ассистент распознаёт intent `manage_availability`
3. Извлечение дат и действия (блок/разблок)
4. Создание/удаление служебного бронирования
5. Подтверждение изменений

**Результат:** Даты заблокированы/разблокированы

---

### 10. Просмотр KPI и статистики

**Описание:** Арендодатель запрашивает аналитику бизнеса.

**Примеры запросов:**
- "Покажи статистику за месяц"
- "Какой доход за последние 3 месяца?"
- "Сколько бронирований в этом году?"
- "Какая загрузка домов?"

**Процесс:**
1. Запрос аналитики
2. Ассистент распознаёт intent `get_dashboard`
3. Запрос к API: `GET /dashboard/owner`
4. Использование Text-to-SQL для сложных запросов
5. Форматирование KPI и графиков текстом

**Результат:** Сводка KPI: бронирования, доход, загрузка

---

### 11. Управление тарифами

**Описание:** Арендодатель просматривает и управляет тарифами.

**Примеры запросов:**
- "Покажи текущие тарифы"
- "Добавь тариф 'Пенсионер' за 100 рублей"
- "Измени цену для взрослого на 300 рублей"

**Процесс:**
1. Запрос информации или изменения
2. Ассистент распознаёт intent `list_tariffs` или `manage_tariff`
3. Запрос к API: `GET /tariffs` или `POST/PATCH /tariffs`
4. Подтверждение изменений

**Результат:** Список тарифов или подтверждение изменения

---

### 12. Учёт расходников

**Описание:** Арендодатель управляет заметками о расходниках.

**Примеры запросов:**
- "Покажи остатки в Старом доме"
- "Добавь заметку: в Старом доме 5 пачек дров"
- "Обнови расходники для Нового дома"

**Процесс:**
1. Запрос информации или добавление
2. Ассистент распознаёт intent `list_consumables` или `add_consumable_note`
3. Запрос к API: `GET /consumable-notes` или `POST /consumable-notes`
4. Форматирование ответа

**Результат:** Список заметок или подтверждение добавления

---

### 13. Text-to-SQL аналитика

**Описание:** Арендодатель задаёт сложные аналитические вопросы.

**Примеры запросов:**
- "Сколько заработано по каждому дому за год?"
- "Какой средний чек бронирования?"
- "Какие месяцы самые загруженные?"
- "Покажи топ арендаторов по количеству поездок"

**Процесс:**
1. Аналитический запрос на естественном языке
2. Ассистент использует Text-to-SQL сервис
3. Генерация SQL-запроса через LLM
4. Выполнение запроса к базе данных
5. Форматирование результатов

**Результат:** Ответ на аналитический вопрос с данными

---

## Общие сценарии

### 14. Справка и помощь

**Описание:** Пользователь получает справочную информацию.

**Примеры запросов:**
- "Как забронировать дом?"
- "Какие есть тарифы?"
- "Как отменить бронирование?"
- "Что делать если задерживаемся?"

**Процесс:**
1. Вопрос о функциональности
2. Ассистент распознаёт intent `help`
3. Ответ из базы знаний или контекста

**Результат:** Инструкция или разъяснение

---

### 15. Голосовое взаимодействие

**Описание:** Пользователь использует голосовой ввод/вывод.

**Функции:**
- Нажатие 🎤 — начало записи
- Повторное нажатие — остановка
- Распознанный текст отправляется как сообщение
- Кнопка 🔊 на сообщениях ассистента — озвучивание

**Применение:** Все сценарии выше доступны через голос

## Техническая реализация

### Intent Mapping

| Intent | Действие | API Endpoint |
|--------|----------|--------------|
| `create_booking` | Создание бронирования | `POST /bookings` |
| `list_bookings` | Список бронирований | `GET /bookings` |
| `cancel_booking` | Отмена бронирования | `PATCH /bookings/{id}` |
| `confirm_booking` | Подтверждение бронирования | `PATCH /bookings/{id}` |
| `list_houses` | Список домов | `GET /houses` |
| `get_house_info` | Информация о доме | `GET /houses/{id}` |
| `check_calendar` | Проверка календаря | `GET /houses/{id}/calendar` |
| `record_stay` | Фиксация результатов | `PATCH /bookings/{id}` |
| `manage_availability` | Управление доступностью | `POST/PATCH /bookings` |
| `get_dashboard` | KPI и статистика | `GET /dashboard/owner` |
| `list_tariffs` | Список тарифов | `GET /tariffs` |
| `manage_tariff` | Управление тарифом | `POST/PATCH/DELETE /tariffs/{id}` |
| `list_consumables` | Список расходников | `GET /consumable-notes` |
| `add_consumable_note` | Добавление заметки | `POST /consumable-notes` |
| `text_to_sql` | Аналитический запрос | `POST /query/natural` |
| `help` | Справка | — |

### Архитектура обработки

```
Пользователь → Сообщение
                    ↓
            [Intent Recognition]
                    ↓
        ┌───────────┼───────────┐
        ↓           ↓           ↓
   Booking    Dashboard    Text-to-SQL
        ↓           ↓           ↓
   API Call    API Call    SQL Generation
        ↓           ↓           ↓
   Response    Response    Query Result
        └───────────┴───────────┘
                    ↓
            [Форматирование ответа]
                    ↓
            Ответ пользователю
```

### API Endpoints для чата

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/chats/user/{user_id}` | Получить или создать чат пользователя |
| GET | `/chats/{chat_id}/messages` | История сообщений |
| POST | `/chats/{chat_id}/messages` | Отправить сообщение |
| POST | `/query/natural` | Text-to-SQL запрос |
