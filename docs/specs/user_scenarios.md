# Пользовательские сценарии

## Обзор

Документ описывает сценарии использования системы бронирования загородного жилья для арендаторов и арендодателей.

---

## Роли

| Роль | Описание |
|------|----------|
| **Арендатор (Tenant)** | Пользователь, бронирующий дом для проживания |
| **Арендодатель (Owner)** | Владелец домов, управляющий бронированиями и календарём |

---

## Сценарии арендатора

### 1. Просмотр списка домов

**Описание:** Арендатор видит все доступные дома с базовой информацией.

**Необходимые данные:**
- House: id, name, description, capacity, is_active

**Результат:** Список активных домов с основными характеристиками.

---

### 2. Просмотр деталей дома

**Описание:** Арендатор открывает карточку дома с полным описанием.

**Необходимые данные:**
- House: все поля

**Результат:** Полная информация о доме: название, описание, вместимость, оснащение.

---

### 3. Просмотр календаря занятости

**Описание:** Арендатор видит занятые и свободные даты для выбранного дома.

**Необходимые данные:**
- Booking: house_id, check_in, check_out, status

**Результат:** Визуальное отображение доступности дома по датам.

---

### 4. Создание бронирования

**Описание:** Арендатор выбирает даты и состав группы для бронирования.

**Необходимые данные:**
- Booking: все поля
- Tariff: для расчёта стоимости

**Процесс:**
1. Выбор дома
2. Выбор дат заезда и выезда
3. Указание состава группы по типам гостей
4. Расчёт предварительной стоимости
5. Подтверждение бронирования

**Результат:** Созданное бронирование со статусом `pending` или `confirmed`.

---

### 5. Просмотр своих бронирований

**Описание:** Арендатор просматривает историю и предстоящие поездки.

**Необходимые данные:**
- Booking: tenant_id, фильтры по датам и статусу

**Результат:** Список бронирований с датами, статусом и суммой.

---

### 6. Отмена бронирования

**Описание:** Арендатор отменяет подтверждённое бронирование.

**Необходимые данные:**
- Booking: id, status

**Результат:** Бронирование переводится в статус `cancelled`, даты освобождаются.

---

### 7. Фиксация результатов поездки

**Описание:** После выезда арендатор указывает фактический состав группы и заметки.

**Необходимые данные:**
- Booking: guests_actual
- StayRecord: фактические данные (опционально)

**Результат:**
- Фиксация фактического количества гостей каждого типа
- Пересчёт итоговой суммы
- Добавление заметок о проживании

---

## Сценарии арендодателя

### 1. Просмотр своих домов

**Описание:** Арендодатель видит список принадлежащих ему домов.

**Необходимые данные:**
- House: owner_id

**Результат:** Список домов с основной информацией и статусом активности.

---

### 2. Управление календарём доступности

**Описание:** Арендодатель закрывает или открывает даты для бронирования.

**Необходимые данные:**
- House: id
- Booking: блокировка дат (создание служебных бронирований)

**Процесс:**
1. Выбор дома
2. Выбор дат для блокировки/разблокировки
3. Подтверждение изменений

**Результат:** Выбранные даты становятся недоступны для бронирования.

---

### 3. Просмотр всех бронирований

**Описание:** Арендодатель просматривает список всех бронирований с фильтрами.

**Необходимые данные:**
- Booking: все поля + фильтры по house_id, датам, статусу

**Фильтры:**
- По дому
- По дате заезда/выезда
- По статусу (pending, confirmed, cancelled, completed)
- По арендатору

**Результат:** Отфильтрованный список бронирований с деталями.

---

### 4. Управление тарифами

**Описание:** Арендодатель создаёт и редактирует тарифы для типов гостей.

**Необходимые данные:**
- Tariff: все поля

**Операции:**
- Создание нового тарифа
- Редактирование существующего
- Удаление неиспользуемого тарифа

**Результат:** Актуальный справочник тарифов для расчёта стоимости.

---

### 5. Контроль остатков расходников

**Описание:** Арендодатель просматривает и обновляет заметки о расходниках в домах.

**Необходимые данные:**
- ConsumableNote: house_id, name, comment

**Операции:**
- Просмотр текущих заметок по дому
- Добавление новой заметки
- Обновление существующей

**Результат:** Актуальная информация об остатках в доме (дрова, продукты и т.д.).

---

## Связи между сущностями

На основе сценариев выделены следующие связи:

```
User 1:N House          (арендодатель владеет домами)
User 1:N Booking        (арендатор создаёт бронирования)
House 1:N Booking       (дом имеет бронирования)
House 1:N ConsumableNote (дом имеет заметки о расходниках)
Tariff используется в Booking (через guests_planned/guests_actual JSON)
```
