# План: Требования к UI и API-контракты

## Цель
Зафиксировать функциональные требования к интерфейсу веб-приложения и дополнить API-контракты новыми endpoint'ами для dashboard, leaderboard, consumable notes и чата.

## Состав работ

1. **ui-requirements.md** — функциональные требования для 4 зон интерфейса:
   - Панель арендодателя (Dashboard)
   - Лидерборд (/leaderboard)
   - Глобальный чат (виджет)
   - Полноэкранный чат (/chat)
   - Механизм авторизации

2. **ui-style-guide.md** — гайд по стилю:
   - Палитра цветов (CSS variables)
   - Типографика
   - shadcn/ui конвенции
   - Tailwind CSS v4 паттерны
   - Accessibility

3. **api-contracts.md** — дополнить существующий файл новыми endpoint'ами:
   - Dashboard KPI
   - Leaderboard stats
   - House stats
   - Consumable notes CRUD
   - Chat endpoints

4. **summary.md** — итоговый документ с результатами
