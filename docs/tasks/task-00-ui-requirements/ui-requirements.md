# Функциональные требования к UI

## Механизм авторизации

### Общий подход
- **Без OAuth и JWT** — простая идентификация через Telegram username
- **Сохранение сессии** — user_id + username в Zustand store с persist middleware (localStorage)
- **Защищённые маршруты** — Route group `(authenticated)` с проверкой наличия auth-данных

### Поток авторизации
1. Пользователь открывает стартовый экран (`/`)
2. Вводит свой Telegram username в поле ввода
3. Frontend выполняет `GET /api/v1/users?telegram_id={username}`
4. Если пользователь найден — сохраняем `user_id`, `username`, `role` в store
5. Если не найден — показываем ошибку, предлагаем обратиться к администратору

### Структура хранения в Zustand
```typescript
interface AuthState {
  userId: number;
  username: string;
  role: 'tenant' | 'owner' | 'both';
  isAuthenticated: boolean;
}
```

## 1. Панель арендодателя (Dashboard)

### Общее описание
Главный экран для арендодателя с KPI, графиками и быстрым доступом к управлению.

### Подстраницы
- `/dashboard` — главная панель с KPI
- `/dashboard/houses` — управление домами
- `/dashboard/bookings` — управление бронированиями
- `/dashboard/tariffs` — управление тарифами
- `/dashboard/consumables` — учёт расходников

### Компоненты

#### KPI-карточки (главная страница)
Отображают ключевые метрики:
- `total_bookings` — общее количество бронирований
- `total_revenue` — общая выручка (руб.)
- `occupancy_rate` — процент загрузки (занятые дни / общие дни)
- `active_bookings` — количество активных бронирований

#### График доходности
- LineChart с данными по месяцам
- Данные из `monthly_revenue[]` (endpoint `/api/v1/dashboard/owner`)

#### Таблица домов (/dashboard/houses)
- CRUD операции: создание, редактирование, удаление
- Колонки: название, вместимость, статус (активен/неактивен), действия
- Модальное окно для создания/редактирования

#### Таблица бронирований (/dashboard/bookings)
- Расширенная таблица с фильтрами:
  - По дому (dropdown)
  - По дате заезда (date range)
  - По статусу (tabs: все / pending / confirmed / completed / cancelled)
- Сортировка по дате создания, дате заезда, сумме
- Действия: просмотр деталей, изменение статуса

#### Управление тарифами (/dashboard/tariffs)
- CRUD операции
- Колонки: название, стоимость, действия
- Валидация: сумма >= 0

#### Учёт расходников (/dashboard/consumables)
- Таблица заметок по домам
- Фильтр по дому
- CRUD операции
- Колонки: дом, категория, описание, дата создания

## 2. Лидерборд (/leaderboard)

### Общее описание
Страница для анализа бронирований с двумя режимами отображения.

### Переключатель режимов (Tabs)
- **Таблица** — расширенная таблица бронирований
- **Календарь** — визуальное отображение занятости

### Фильтры (общие для обоих режимов)
- По дому (multi-select)
- По дате (date range)
- По статусу (multi-select)

### Режим Таблица
- Расширенная версия таблицы из Dashboard
- Дополнительные колонки: длительность проживания, стоимость за ночь
- Экспорт в CSV

### Режим Календарь
- **Custom CSS Grid** вместо сторонних библиотек
- **Строки** — дома (House name)
- **Столбцы** — дни месяца
- **Ячейки** — бронирования с цветовой индикацией:
  - `confirmed` — default цвет (основной)
  - `pending` — secondary цвет
  - `completed` — outline стиль
  - `cancelled` — не отображается
- Навигация по месяцам (prev/next)
- Tooltip при наведении: даты, гости, сумма

### График доходности
- LineChart под календарём/таблицей
- Данные из `/api/v1/dashboard/leaderboard`

### Экспорт CSV
- Кнопка "Экспорт" в правом верхнем углу
- Формат: booking_id, house_name, check_in, check_out, status, total_amount

## 3. Глобальный чат (виджет)

### Общее описание
Плавающий виджет чата, доступный на всех экранах внутри authenticated layout.

### Компоненты

#### Плавающая кнопка
- Позиция: bottom-right (fixed)
- Иконка: MessageCircle
- Badge с количеством непрочитанных (опционально для MVP)

#### Sheet (shadcn/ui)
- `side="right"`
- Ширина: 400px (desktop), 100vw (mobile)
- Заголовок: "Чат с помощником"

#### Логика работы
- **Polling** для обновлений (каждые 5 секунд) — MVP-решение
- **Автоприветствие** при создании чата (первое сообщение от бота)
- **Оптимистичное обновление** — сообщение появляется сразу, статус меняется после ответа сервера

#### Сообщения
- Группировка по дате (Separator + Badge)
- Аватар: бот (Bot icon), пользователь (User icon)
- Время отправки
- ScrollArea с автопрокруткой к последнему сообщению

## 4. Полноэкранный чат (/chat)

### Общее описание
Отдельная страница для детального общения с LLM-ассистентом.

### Layout

#### Двухколоночный (desktop)
- **Левая колонка** (280px): список диалогов
- **Правая колонка**: активный чат

#### Одноколоночный (mobile)
- Список диалогов — отдельный экран
- Чат — отдельный экран с кнопкой "Назад"

### Левая панель (список диалогов)
- **Поиск** — фильтр диалогов по содержимому
- **Список диалогов**:
  - Avatar + AvatarFallback (первая буква названия)
  - Название диалога (автогенерируемое или пользовательское)
  - Последнее сообщение (preview)
  - Дата последнего сообщения
- **Кнопка "Новый чат"** — создаёт новый диалог

### Основная область (активный чат)
- **ScrollArea** для сообщений
- **Группировка по дате**:
  ```
  ─────── 4 апреля 2026 ───────
  [сообщение 1]
  [сообщение 2]
  ─────── Сегодня ───────
  [сообщение 3]
  ```
- **InputGroup**:
  - Textarea с авторасширением
  - Отправка по Ctrl+Enter
  - Кнопка отправки (Enter без Ctrl — новая строка)

### Состояния
- Пустое состояние (нет диалогов) — призыв к действию
- Загрузка — skeleton
- Ошибка — retry button

## Навигация

### Структура маршрутов

```
/                           # Стартовый экран (ввод username)
├── (authenticated)/        # Защищённая группа
│   ├── dashboard/          # Панель арендодателя
│   │   ├── page.tsx        # KPI и графики
│   │   ├── houses/         # Управление домами
│   │   ├── bookings/       # Управление бронированиями
│   │   ├── tariffs/        # Управление тарифами
│   │   └── consumables/    # Учёт расходников
│   ├── leaderboard/        # Лидерборд
│   └── chat/               # Полноэкранный чат
```

### Навигационное меню (authenticated)
- Dashboard (для owner/both)
- Leaderboard (для owner/both)
- Chat
- Выход (очистка store)
